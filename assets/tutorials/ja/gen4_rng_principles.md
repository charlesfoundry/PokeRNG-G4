# 第四世代乱数の原理

第四世代の乱数は、次のように考えると分かりやすいです。

```text
本体時刻 + Delay で初期 seed が決まる
その seed から続く固定の乱数列が結果を決める
プレイヤー操作、NPC、電話、ペラップなどが乱数列を消費する
```

実機での目標は次の3つです。

1. 正しい `seed` を命中する。
2. 正しい `frame / advance` まで消費する。
3. 正しい位置でポケモン生成を発生させる。

## 基本用語

| 名称 | 簡単な意味 | 実機での意味 |
| --- | --- | --- |
| `seed` | 乱数列の開始点 | 同じ seed なら後続の乱数列も同じ |
| `Delay` | ゲーム起動から seed 読み込みまでのハードウェア的なずれ | 目標 seed を命中するための重要な値 |
| `second` | seed 生成に使われる秒 | DS 時刻は分までしか設定できないため、秒は待って合わせる |
| `frame / advance` | RNG が何回消費されたか | seed 後のどの結果を使うかを決める |
| `PID` | 隠し人格値 | 性格、性別、特性スロット、色違いなどを決める |
| `IV` | 個体値 | 通常は PID 後の RNG 結果から決まる |
| `TID/SID` | 表ID / 裏ID | PID と組み合わせて色違い判定に使われる |

## seed は時刻からどう決まるか

Gen4 の初期 seed は、DS 本体時刻と Delay から作られます。

簡略化すると：

```text
AB = 月 * 日 + 分 + 秒
CD = 時

seed = (AB << 24) | (CD << 16)
seed = seed + Delay + 年の補正
```

つまり：

```text
年月日時分秒  seed の上位部分を決める
Delay         seed の下位部分を決める
年            表示される Delay を補正する
```

同じ seed に複数の日時候補が存在するのは、別の日付、分、秒でも同じ上位部分を作れる場合があるためです。

> プレイヤーにとって重要なのは日付そのものではなく、その日時と Delay で目標 seed が作れるかどうかです。

## Delay とは

`Delay` は次のように考えられます。

```text
ゲームを起動してから、
ゲームが初期 seed を読み込むまでに発生する
ハードウェア的なタイミングのずれ。
```

通常の実機操作では、近似的に次のように考えられます。

```text
DS 本体メニューで A を押してゲームを起動してから、
レポート選択画面で A を押してレポートに入るまでの
操作時間によって生じる起動時のずれ。
```

通常のゲーム内フレームではなく、待ち時間の秒数でもありません。

たとえば目標が：

```text
Seed: 00130AE9
Delay: 2782
Second: 57
```

と表示されている場合、意味は：

```text
目標時刻の57秒で、
Delay 2782 の起動ずれでゲームに入り、
seed 00130AE9 を得る。
```

タイマーは、この操作を安定させるために使います。

## 校準が必要な理由

プレイヤーが毎回最初から目標 Delay を正確に命中することはできません。

通常の流れは：

```text
目標 seed / Delay / second を選ぶ
タイマーでゲームに入る
電話、コイントス、実際の捕獲結果などで命中を確認する
実際の Delay を逆算する
校準を適用する
もう一度試す
```

例：

```text
目標 Delay: 2782
実際の命中: 2772
```

App はこのずれを使って校準 Delay を更新し、次回のタイマーをプレイヤーの実際の入力感覚に近づけます。

## 年と Delay の奇偶

Gen4 の seed 計算には年の補正が含まれます。

```text
Delay + year - 2000
```

そのため、年は Delay の奇偶に影響します。

たとえば：

```text
目標 Delay は偶数
実際の命中は常に奇数
```

またはその逆の場合、通常は App の検索条件を変える必要はありません。DS 本体の年を +1 または -1 することで、奇偶を切り替えられます。

## seed の後に起きること

ゲームが初期 seed を得ると、固定の式で次の乱数を作り続けます。

```text
seed = seed * 0x41C64E6D + 0x6073
```

乱数を1回作るたびに、RNG が1回消費されたと考えます。

したがって：

```text
同じ seed + 同じ消費回数 = 同じ結果
```

出現スロット、レベル、PID、性格、性別、特性スロット、個体値、持ち物、一部のフォルムなどは、後続の乱数から読み取られます。

## 電話、コイントス、ペラップ

### HGSS 電話

HGSS の電話は seed 確認に使え、同時に RNG を消費します。

ウツギ博士またはジャグラーに電話すると、`E/K/P` のいずれかになります。App は観測した電話列から次を逆算できます。

```text
どの seed がこの E/K/P 列を出すのか？
この電話列が始まる前、現在何フレーム目だったのか？
```

### DPPt コイントス

DPPt のコイントスは、seed 後の乱数列を観測するために使います。主な用途は seed 確認です。

### ペラップ

ペラップは主にフレーム消費に使います。

録音した鳴き声を1回聞くたびに：

```text
RNG を1回消費する
プレイヤーは1フレーム進む
その乱数で音程が決まる
```

## Method 1 / J / K

Gen4 のすべての対象が同じ順序で乱数を読むわけではありません。

| Method | 主な用途 | 説明 |
| --- | --- | --- |
| Method 1 | 御三家、贈り物、一部の特殊生成 | 比較的直接的な読み取り順 |
| Method J | DP/Pt の固定シンボルや野生など | シンクロ、性格選択、出現スロットなどを処理する |
| Method K | HGSS の固定シンボルや野生など | HGSS 側の Method J に相当する処理 |

同じ初期 seed と LCRNG を使いますが、乱数を読む順番が異なります。

そのため、同じ seed、同じフレームでも、Method が違えば結果はまったく異なる場合があります。

## 野生エンカウントが複雑な理由

野生エンカウントでは、PID と IV だけを生成するわけではありません。多くの場合、次も計算されます。

- エンカウントが発生するか
- 出現スロット
- レベル
- 種族
- 性格と PID
- 個体値
- 持ち物
- 一部の特殊フォルム

先頭特性も処理に影響します。

| 先頭特性 | 影響する可能性 |
| --- | --- |
| シンクロ | 性格 / PID 生成 |
| メロメロボディ | PID の候補範囲と性別 |
| せいでんき / じりょく | 特定タイプの出現 |
| プレッシャー | レベル |
| ふくがん | 持ち物 |

App の先頭設定は、実際のゲーム状態と一致している必要があります。

## ID乱数

ID乱数も時刻 seed によって決まります。ただし TID/SID は通常の LCRNG から直接生成されるのではなく、seed から MT を起動して生成されます。

簡略化すると：

```text
時刻 + Delay -> seed
seed -> MT -> TID/SID
```

TID/SID が重要なのは、色違い判定に次が使われるためです。

```text
TID ^ SID ^ PID上位 ^ PID下位
```

TID/SID を制御できれば、一部の優秀な PID を色違いにできます。

## タマゴ乱数が2段階に分かれる理由

Gen4 のタマゴ乱数は2段階です。

```text
タマゴ生成
タマゴ受取
```

タマゴ生成で決まるもの：

- PID
- 性格
- 性別
- 特性スロット
- 色違いかどうか

タマゴ受取で決まるもの：

- 個体値
- 親からの遺伝

そのため、タマゴが生成済みでまだ受け取っていない状態なら、リセットして受取段階の個体値を乱数できます。

## よくある間違い

### 1. Delay を frame と同じものとして扱う

`Delay` は起動時のタイミングのずれです。
`frame / advance` は seed の後に RNG が何回消費されたかです。

この2つは同じものではありません。

### 2. seed は合っているが frame が違う

主な原因：

- NPC の向き変更や移動
- 徘徊ポケモンの初期消費
- 電話回数と App の入力回数が一致していない
- ペラップの回数を数え間違えた
- エンカウント発生タイミングの理解違い

### 3. 検索条件と実機状態が一致していない

例：

- App ではシンクロなし、実機では先頭がシンクロ
- 時間帯が違う
- ダブルスロットの状態が違う
- 場所やエンカウント方法が違う
- レベル範囲や出現スロットの想定が違う

## 一言でまとめると

```text
Gen4 乱数は運任せではありません。
DS 時刻と Delay で seed を命中し、
制御できる操作で RNG を目標位置まで消費する方法です。
```
