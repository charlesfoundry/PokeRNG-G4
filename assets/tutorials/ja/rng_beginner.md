# 乱数とは

乱数は、ゲームを改造することではありません。セーブデータにポケモンを直接書き込むことでもありません。

乱数で行うことは、次の考え方に近いです。

```text
ゲーム本来のランダム処理を調べ、
正しい時刻と正しい操作で、
ゲーム自身に目標の結果を生成させる。
```

## ゲーム内の「ランダム」

ゲーム内では、多くの結果がランダムに見えます。

- どのポケモンに出会うか
- 性格
- 個体値
- 色違いかどうか
- 性別
- 特性スロット
- タマゴの結果

しかし、ゲーム機は本当の意味で完全なランダムを作っているわけではありません。`seed` と呼ばれる開始点から、決まった計算式で次の結果を順番に作っています。

長い結果の列が、すでに順番を持って並んでいると考えると分かりやすいです。

```text
1番目の結果
2番目の結果
3番目の結果
……
```

普通に遊んでいる時は、自分が何番目の結果に到達しているのか分からないため、ランダムに見えます。

乱数は、この結果の列を計算し、欲しい位置に到達するための操作です。

## ある瞬間を再現する

たとえば、ある日、ある時刻にゲームを起動し、次のような操作をしたとします。

```text
DSの時刻を設定する
ゲームを起動する
レポートを読み込む
メニューを開く
数歩歩く
Aを押してエンカウントを発生させる
```

その結果、1匹のポケモンに出会いました。

ゲームの状態が完全に同じで、さらに次の条件を再現できた場合、

- 同じ DS 時刻
- 同じタイミングでゲームに入る
- 同じ操作順
- 同じ操作回数
- 同じタイミングでエンカウントを発生させる

理論上、ゲームは同じ乱数列の同じ位置に到達し、非常に近い、または完全に同じ結果を生成します。

> 乱数の核心は、結果を作り変えることではありません。特定の時刻と操作手順を再現することです。

## App がしていること

プレイヤーが自力で次の情報を知るのは非常に困難です。

```text
どの日
何時何分何秒
どの Delay
何フレーム目
で目標が出るのか
```

App は、これらの条件を計算します。

App が示すのは次のような内容です。

- DS 時刻をいつに設定するか
- いつゲームに入るか
- 何フレームまで消費するか
- いつ A を押す、またはエンカウントを発生させるか

つまり：

```text
乱数ツールは結果を書き換えるものではありません。
「その日、その秒、その操作手順」を探すものです。
```

## 乱数の目的

普通に遊んでいても、次のような結果は自然に出る可能性があります。

- 色違い
- 良い性格
- 高個体値
- 特定の性別
- 特定の特性スロット
- 理想のタマゴ
- 特定の TID/SID

ただし、確率が低いことが多いです。

たとえば同時に、

```text
色違い + 良い性格 + 高個体値 + 目標ポケモン
```

を狙う場合、完全に運だけで待つと非常に時間がかかります。

乱数の目的は、

```text
何度も試して運を待つ
```

という作業を、

```text
目標位置を計算する
タイミングを練習する
結果を安定して再現する
```

という作業に変えることです。

## なぜ正規の結果なのか

乱数で得たポケモンは、ゲーム自身が生成したものです。

その過程では：

- セーブデータを改造しない
- ポケモンを注入しない
- メモリを改変しない
- PID や個体値を強制的に書き込まない
- ゲーム本来の生成規則を無視しない

プレイヤーが制御しているのは外部条件だけです。

- DS 本体の時刻
- ゲームに入るタイミング
- ボタン入力のタイミング
- 電話の回数
- ペラップの鳴き声の回数
- エンカウントやタマゴ受取のタイミング

最後にポケモンを生成しているのは、元のゲームプログラムです。

そのため、データは自然にゲームの規則に従います。

| データ | 正規である理由 |
| --- | --- |
| PID | ゲームの RNG が生成する |
| 個体値 | ゲームの RNG が生成する |
| 性格 | PID または RNG 結果から決まる |
| 色違い | TID/SID/PID によって通常通り判定される |
| 場所とレベル | ゲーム本来の出現データに基づく |

## 改造との違い

改造では通常、次のようなことを行います。

```text
欲しいデータをセーブに直接書き込む。
またはメモリを変更して、結果を目的の値にする。
```

乱数では：

```text
データを改造しない。
セーブを改造しない。
時刻と操作によって、ゲーム自身に目標を生成させる。
```

最大の違いは次の点です。

```text
改造は「データを書き換えて作る」。
乱数は「ゲーム自身に生成させる」。
```

## 一言でまとめると

普通のプレイが、どこに当たりがあるか分からない結果の列から引くことだとすれば、乱数は：

```text
目標の結果がどこにあるかを計算し、
正しい時刻と操作を再現し、
ゲーム自身にその結果を引かせること。
```
