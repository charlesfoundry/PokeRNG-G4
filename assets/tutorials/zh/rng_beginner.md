# 什么是乱数

乱数不是修改游戏，也不是把一只宝可梦强行写进存档。

乱数做的事情更接近：

```text
研究游戏自己的随机规律，
然后在正确的时间、用正确的操作，
让游戏自然生成目标结果。
```

## 游戏里的“随机”

游戏里很多内容看起来是随机的：

- 遇到哪只宝可梦
- 性格是什么
- 个体值是多少
- 是否闪光
- 性别是什么
- 特性是哪一个
- 蛋孵出来是什么结果

但游戏机不会产生真正意义上的完全随机。它会根据一个起点，也就是 `seed`，按固定公式一步一步生成后续结果。

可以把它想成一条已经排好的结果队列：

```text
第1个结果
第2个结果
第3个结果
……
```

普通游玩时，玩家不知道自己会走到第几个结果，所以感觉是随机的。

乱数就是把这条结果队列算出来，然后想办法走到想要的位置。

## 复现某个时间点

假设你在某一天、某个具体时间打开游戏，并做了一套操作：

```text
设置DS时间
启动游戏
进入存档
打开菜单
走几步
按A触发遇敌
```

最后你遇到了一只宝可梦。

如果游戏状态完全一样，并且你能做到：

- 同一个 DS 时间
- 同一个进入游戏时机
- 同样的操作顺序
- 同样的操作次数
- 同样的触发时机

那么理论上，游戏就会再次走到同一条随机序列上的同一个位置，生成非常接近甚至完全相同的结果。

> 乱数的核心不是创造结果，而是复现某个时间点和某套操作路径。

## App 在帮你做什么

玩家自己很难知道：

```text
哪一天
几点几分几秒
什么Delay
第几帧
会出现目标宝可梦
```

App 的作用是帮你计算这些条件。

它会告诉你：

- DS 时间应该设成什么时候
- 应该在什么时候进入游戏
- 应该推进到第几帧
- 应该什么时候按 A 或触发遇敌

换句话说：

```text
乱数工具不是帮你修改结果，
而是帮你找出“那一天、那一秒、那套操作”。
```

## 乱数的目的

正常游玩时，也可能自然遇到：

- 闪光宝可梦
- 好性格
- 高个体值
- 指定性别
- 指定特性
- 理想的蛋
- 特定 TID/SID

只是概率通常很低。

如果同时要求：

```text
闪光 + 好性格 + 高个体值 + 指定宝可梦
```

纯靠运气可能需要非常久。

乱数的目的就是把：

```text
反复碰运气
```

变成：

```text
计算目标位置
练习命中时机
稳定复现结果
```

## 为什么是合法的

通过乱数获得的宝可梦，是游戏自己生成的。

整个过程中：

- 没有修改存档
- 没有注入宝可梦
- 没有改内存
- 没有强行写入 PID 或个体值
- 没有绕过游戏的生成规则

玩家只是控制外部条件：

- DS 机器时间
- 进入游戏的时机
- 按键时机
- 电话次数
- 喋喋不休次数
- 触发遇敌或领取蛋的时机

最后生成宝可梦的，仍然是原版游戏自己的程序。

所以它的数据会自然符合游戏规则：

| 数据 | 为什么合法 |
| --- | --- |
| PID | 由游戏 RNG 生成 |
| 个体值 | 由游戏 RNG 生成 |
| 性格 | 由 PID 或 RNG 结果决定 |
| 闪光 | 按 TID/SID/PID 正常判定 |
| 地点和等级 | 来自游戏原本的遇敌表 |

## 和作弊修改的区别

作弊修改通常是：

```text
直接把想要的数据写进存档，
或者修改游戏内存，让结果变成目标值。
```

乱数则是：

```text
不改数据，
不改存档，
只通过时间和操作，
让游戏自然生成目标结果。
```

最大的区别是：

```text
作弊修改是“改出一只宝可梦”。
乱数是“让游戏自己生成那只宝可梦”。
```

## 一句话总结

如果说普通游玩是在不知道结果队列的情况下抽签，那么乱数就是：

```text
先算出目标签在哪，
再复现正确的时间和操作，
让游戏自己抽到那一签。
```
